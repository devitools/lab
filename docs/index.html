lab.devi.tools

URL pública pro seu projeto local em 1 clique.

O floofy.exe é um appzinho que roda no seu Windows e transforma um projeto que tá só na sua máquina numa URL que qualquer amigo pode abrir no navegador, com HTTPS e tudo. Dois jeitos de usar:

⬇ Baixar floofy.exe Windows · ~15 MB · single-file, sem instalar nada

📤 Publicar pasta

  1. Rodar npm run build
  2. Abrir o floofy.exe
  3. Escolher a pasta dist/
  4. Clicar Publicar
  5. Copiar a URL e mandar pro amigo
Funciona pra sempre (até apagar) — seu PC pode tá desligado que continua no ar.

🔌 Conectar porta

  1. Rodar npm run dev
  2. Abrir o floofy.exe
  3. Digitar a porta (5173 pro Vite)
  4. Clicar Conectar
  5. Copiar a URL e mandar pro amigo
Funciona enquanto o floofy.exe tiver aberto. Fechou = caiu. Mas mostra mudanças ao vivo (HMR).

Tô só com HTML/CSS/JS (sem build)

Vale tanto pra projeto com build (Vite/React/Vue/etc.) quanto pra HTML escrito na mão. No caso sem build:

📤 Publicar pasta

Pula o npm run build. Aponta direto pra pasta do projeto que tem o index.html dentro. O floofy zipa e sobe.

Funciona se index.html tá na raiz da pasta escolhida e os caminhos pra CSS/JS são relativos (ex: ./style.css, não /style.css com barra inicial).

🔌 Conectar porta

Precisa de um servidorzinho local rodando o HTML. Três opções:

  1. WebStorm built-in (porta 63342) — abre o HTML pelo botão browser, mas antes habilitar em Settings → Build, Execution, Deployment → Debugger → "Can accept external connections"
  2. VSCode Live Server (porta 5500) — extensão "Live Server" da Microsoft, botão "Go Live" embaixo
  3. Python direto (porta sua escolha) — na pasta do projeto: python -m http.server 5500
WebStorm built-in tem peculiaridade: a URL incluí o nome do projeto no caminho (ex: localhost:63342/meu-site/index.html). No floofy fica <slug>.devi.tools/meu-site/. Se quiser URL limpa (sem o nome do projeto no caminho), prefira o python -m http.server.

A pegadinha mais comum

"Blocked request. This host is not allowed."
Vite (e maioria dos dev servers) recusa Host externo por padrão. Se isso aparecer no navegador no modo conectar porta, ajusta o vite.config.js: 
export default {
  server: {
    host: true,
    allowedHosts: ['.devi.tools'],
  },
}

Outros frameworks: CRA → HOST=0.0.0.0 DANGEROUSLY_DISABLE_HOST_CHECK=true npm start. Next.js → next dev -H 0.0.0.0.
