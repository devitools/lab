URL pública pro seu projeto local em 1 clique

Abra o lab no seu computador, aponte uma pasta ou uma porta, e receba um link HTTPS pra mandar pro amigo. Sem SSH, sem ngrok, sem cadastro.

Windows · lab.exe macOS · lab.app

Single-file ~15MB. Apple Silicon. Sem dependência externa.

Dois cliques

Sem terminal, sem chave SSH, sem token de cadastro. Abriu, escolheu, clicou.

HTTPS automático

Wildcard *.devi.tools com certificado renovado sozinho. URL pronta pra mandar.

Open source

Cliente Python, servidor Go, tudo no GitHub. Sem rastreamento, sem analytics, sem firula.

Como usar

Dois modos, mesma simplicidade

Publicar uma pasta de build pronto, ou conectar uma porta local ao vivo.

Publicar pasta

Build estático · URL permanente

  1. Abrir o lab
  2. Escolher a pasta do projeto (com index.html dentro)
  3. Clicar Publicar
  4. Copiar a URL e mandar pro amigo
Funciona pra sempre (até apagar). Seu computador pode tá desligado que continua no ar.

Conectar porta

Dev server ao vivo · URL temporária

  1. Subir um servidor local. Três opções:
    • WebStorm built-in (porta 63342) — habilitar em Settings → Build → Debugger → "Can accept external connections"
    • VSCode extensão Live Server (porta 5500)
    • Python python -m http.server 5500
  2. Abrir o lab e digitar a porta
  3. Clicar Conectar
  4. Copiar a URL e mandar pro amigo
Funciona enquanto o lab tiver aberto. Fechou = caiu. Mas mostra mudanças ao vivo.

Comparação

Qual modo escolher?

Depende se você quer um link que sobreviva ou que mostre mudanças ao vivo.

Publicar pasta Conectar porta
Funciona com o computador desligado sim não
Mostra mudanças ao vivo não sim
Bom pra entregar trabalho muito instável
Bom pra demo rápida muito muito
Funciona com backend (Java, PHP…) só estático se tem porta

Projeto com build

Vite, React, Vue, Next…

Mesma coisa, só que com um passo extra de npm.

Publicar + build

Pra entregar trabalho

  1. Rodar npm run build
  2. No lab, escolher a pasta dist/ (ou build/)
  3. Publicar normal

Conectar dev server

Pra mostrar mudanças ao vivo

  1. Rodar npm run dev
  2. Porta: Vite 5173, CRA 3000, Next 3000
  3. No lab, digitar essa porta e Conectar

Atenção

Pegadinhas comuns

"Blocked request. This host is not allowed."
Vite e similares rejeitam Host externo por padrão. Ajustar o vite.config.js: 
export default {
  server: {
    host: true,
    allowedHosts: ['.devi.tools'],
  },
}

CRA: HOST=0.0.0.0 DANGEROUSLY_DISABLE_HOST_CHECK=true npm start
Next.js: next dev -H 0.0.0.0

WebStorm coloca o nome do projeto no caminho
Ex.: localhost:63342/meu-site/index.html vira <slug>.devi.tools/meu-site/. Se quer URL na raiz, prefira python -m http.server.
