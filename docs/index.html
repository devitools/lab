lab.devi.tools

URL pública pro seu projeto local em 1 clique.

O lab.exe é um appzinho que roda no seu Windows e transforma um projeto que tá só na sua máquina numa URL que qualquer amigo pode abrir no navegador, com HTTPS e tudo. Dois jeitos de usar:

⬇ Windows · lab.exe ⬇ macOS · lab.app Windows: single-file, dois cliques. macOS (Apple Silicon): descompactar e mover pra Aplicativos. Na 1ª abertura, click-direito no lab.app → Abrir (Gatekeeper bloqueia apps não assinados).

Como usar

📤 Publicar pasta

  1. Abrir o lab
  2. Escolher a pasta do projeto (com index.html dentro)
  3. Clicar Publicar
  4. Copiar a URL e mandar pro amigo
Funciona pra sempre (até apagar) — seu computador pode tá desligado que continua no ar.

🔌 Conectar porta

  1. Rodar um servidor local na sua pasta. Três opções:
    • WebStorm: built-in na porta 63342 (antes habilitar em Settings → Build, Execution, Deployment → Debugger → "Can accept external connections")
    • VSCode: extensão Live Server (porta 5500, botão "Go Live")
    • Python: na pasta do projeto rodar python -m http.server 5500
  2. Abrir o lab
  3. Digitar a porta
  4. Clicar Conectar
  5. Copiar a URL e mandar pro amigo
Funciona enquanto o lab tiver aberto. Fechou = caiu. Mas mostra mudanças ao vivo.

Projeto com build (Vite, React, Vue, Next)?

Mesma coisa, com um passo extra de npm:

📤 Publicar + build

  1. Rodar npm run build
  2. No lab, escolher a pasta dist/ (ou build/) gerada
  3. Publicar normal

🔌 Conectar dev server

  1. Rodar npm run dev
  2. Porta varia: Vite 5173, CRA 3000, Next 3000
  3. No lab, digitar essa porta e Conectar

Pegadinhas

"Blocked request. This host is not allowed."
Vite e similares rejeitam Host externo por padrão. Se aparecer no navegador no modo conectar porta, ajustar o vite.config.js: 
export default {
  server: {
    host: true,
    allowedHosts: ['.devi.tools'],
  },
}

Outros: CRA → HOST=0.0.0.0 DANGEROUSLY_DISABLE_HOST_CHECK=true npm start. Next.js → next dev -H 0.0.0.0.

WebStorm built-in coloca o nome do projeto no caminho (ex: localhost:63342/meu-site/index.html). No lab a URL fica <slug>.devi.tools/meu-site/. Se quiser sem esse prefixo, prefira python -m http.server.

Exemplo de URL gerada

O nome amigável é opcional. Sempre vem com 6 letrinhas aleatórias no final pra ninguém adivinhar: 

https://meu-tcc-h7k2nq.devi.tools

Slugs estáticos sem acesso por 30 dias são apagados automaticamente pra economizar espaço.

Código aberto em github.com/devitools/lab. Servidor: lab (Go). Cliente: lab.exe (Python + Tkinter).
